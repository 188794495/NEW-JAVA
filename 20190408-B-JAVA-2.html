1.背景介绍

什么是Mybatis?

mybatis是一个优秀的基于java的持久层框架，它内部封装了jdbc，使开发者只需要关注sql语句本身，而不需要花费精力去处理加载驱动、创建连接、创建statement等繁杂的过程。 mybatis通过xml或注解的方式将要执行的各种statement配置起来，并通过java对象和statement中sql的动态参数进行映射生成最终执行的sql语句，最后由mybatis框架执行sql并将结果映射为java对象并返回。

2.知识剖析

Mybatis的常用标签

insert

属性介绍:id :唯一的标识符,parameterType:传给此语句的参数的全路径名或别名 例:cn.ki.pojo.User

trim

属性介绍:trim是更灵活的去处多余关键字的标签，他可以实践where和set的效果。

if

属性介绍:if标签通常用于WHERE语句中，通过判断参数值来决定是否使用某个查询条件, 他也经常用于UPDATE语句中判断是否更新某一个字段，还可以在INSERT语句中用来判断是否插入某个字段的值

select

属性介绍:id :唯一的标识符.parameterType:传给此语句的参数的全路径名或别名 例:com.test.poso.User或userresultType :语句返回值类型或别名。注意，如果是集合，那么这里填写的是集合的泛型，而不是集合本身（resultType 与resultMap 不能并用）

sql

属性介绍:sql标签相当于java的变量,将一些繁杂的字段写在标签中,在其他地方使用include标签插入

include

属性介绍:include标签用于引用sql标签,可以在sql语句内插入此标签,很方便

delete

属性介绍:id :唯一的标识符,parameterType:传给此语句的参数的全路径名或别名 例:cn.ki.pojo.User

foreach

属性介绍:foreach标签主要用于构建in条件,他可以在sql中对集合进行迭代。

choose

属性介绍:从多个选项中选择一个,按顺序判断when中的条件出否成立，如果有一个成立，则choose结束。当choose中所有when的条件都不满则时，则执行 otherwise中的sql.类似于Java 的switch 语句,choose为switch，when为case，otherwise则为default

update

属性介绍:id :唯一的标识符,parameterType:传给此语句的参数的全路径名或别名 例:cn.ki.pojo.User

set

属性介绍:set标签可以将动态的配置SET 关键字，和剔除追加到条件末尾的任何不相关的逗号。

4.常见问题

where标签

属性介绍:where标签会知道如果它包含的标签中有返回值的话，它就插入一个‘where’。此外，如果标签返回的内容是以AND 或OR 开头的，则它会剔除掉。

4.常见问题

trim标签

属性介绍:trim是更灵活的去除多余关键字的标签，他可以实践where和set的效果. 属性“prefix”表示：加入前缀where,属性“prefixOverrides”表示：自动覆盖第一个“and”或者“or”,后缀的用法类似.

4.常见问题

set标签

属性介绍:set标签可以将动态的配置SET 关键字，和剔除追加到条件末尾的任何不相关的逗号。
