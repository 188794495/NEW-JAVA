resultMap和resultType的区别

resultType：当使用resultType做SQL语句返回结果类型处理时，对于SQL语句查询出的字段在相应的实体类中必须有和它相同的字段对应，而resultType中的内容就是实体类在项目中的位置。

resultMap：当使用resultMap做SQL语句返回结果类型处理时，需要在mapper.xml中定义resultMap进行实体类和相应表字段的对应。也就是说,我们自己定义数据库和实体类字段名的映射

mybatis中#{}和${}的区别

#{ }：占位符号，好处防止sql注入,一个 #{ } 被解析为一个参数占位符 ? 。

${ }：sql拼接符号,${ } 仅仅为一个纯碎的 string 替换，在动态 SQL 解析阶段将会进行变量替换。

#{}方式一般用于传入字段值，并将该值作为字符串加到执行sql中，一定程度防止sql注入；

${}方式一般用于传入数据库对象，例如传入表名，不能防止sql注入，存在风险。

两者都是动态的向sql语句中传入需要的参数

eg:select id,name,age from student where name=#{name} 这个name是动态的，可变的。当你传入什么样的值，就会根据你传入的值执行sql语句。

传入一个不改变的字符串或者传入数据库字段(列名)，例如要传入order by 后边的参数这种情况下必须使用${}。

什么是SQL注入?

通过把这个命令插入到表单提交的查询字符串恶意执行的命令

举例: 正常来说我们在登录的时候,就会输入用户名和密码,只有输入用户名和密码以后,后台程序查一遍数据库,能拿到正确的信息才会让你登录,但是如果你输入一些不合法的字符串,就会直接进入网站内部. 比如,直接在用户名一栏什么也不输,在密码里输入 'or 1=1#

SELECT * FROM `user` where name= '' and password = ''or 1=1#' ;

防止sql注入的方法:

1.使用预编译绑定变量的SQL语句 2.严格加密处理用户的机密信息 3.不要随意开启生产环境中Webserver的错误显示 4.使用正则表达式过滤传入的参数 5.字符串过滤 6.检查是否包函非法字符
