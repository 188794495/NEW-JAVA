1.背景介绍

类是属性和方法容器，但是这些属性和方法没有任何区别的堆积在一起，当开发人员要寻找某个方法时，将很不方便，而且直接使用实现类会导致父类-子类间出现强耦合，很容易导致程序失控，而接口的出现就是为了解决这个问题。

2.知识剖析

2.1 什么是接口？

接口的概念较为抽象，可以理解为接口是一系列方法和特征的集合，通常用interface来声明。方法和特征是类的组成部分，如果从编程的角度来讲，Java的接口可以对应Java中特定的语法结构。Java接口与Java抽象类是不一样的，抽象类可以实现某些方法，而接口则不能，一个类需要通过继承接口的方式，从而来继承接口的抽象方法。

2.2 接口的特点

（1）接口不可以被实例化，但是可以被实现。一个实现接口的类，必须实现接口内所描述的所有方法，否则就必须声明为抽象类。

（2）接口里面的所有常量都是静态常量，为 public static final 修饰； 可以通过类命名直接访问：ImplementClass.name； 不过不推荐使用接口来创建常量。

（3）接口中可以含有变量，但是接口中的变量会被隐式的指定为 public static final 变量（并且只能是 public，用 private 修饰会报编译错误）。

（4）实现类可以实现多个接口（java多继承）。

2.3 接口的作用

（1）重要性：在java中，abstract class和interface是支持抽象类定义的两种机制，赋予了Java面向对象的能力

（2）简单、规范性：大型项目，通过定义接口可以告诉开发人员需要实现的业务，同时限制了命名规范。

（3）维护、扩展性。接口定义了功能，然后通过定义类来实现功能。这样需要使用接口的功能时就是使用这个接口去引用实现它的类。 这样当接口功能需要更新时，对应功能上来讲，只需要更换引用的实现类。

（4）安全、严密性：接口是实现程序松耦合的重要手段，它描述了系统对外的所有功能，而不涉及任何具体的实现细节。

2.4 抽象类

（1）抽象类：在面向对象的概念中，所有的对象都是通过类来描绘的，但是反过来，并不是所有的类都是用来描绘对象的， 如果一个类中没有包含足够的信息来描绘一个具体的对象，这样的类就是抽象类。

（2）抽象类不可被实例化，所以抽象类必须被继承，才能被使用，抽象类中可以没有抽象方法。

（3）抽象方法只保留方法的功能，而具体的执行，交给继承抽象类的子类，由子类重写抽象方法； 抽象方法必须为 public 或者 protected（因为如果为 private，则不能被子类继承，子类便无法实现该方法），缺省情况下默认为 public

（4）若子类继承抽象类，并重写了所有的抽象方法，则此类是一个“实体类”，即可以实例化

（5）若子类继承抽象类，没有重写所有的抽象方法，意味着此类中仍有抽象方法，则此类必须声明为抽象的。

3.常见问题

3.常见问题

3.1 接口与抽象类的共同点

（1）都是抽象的，没有具体的实现方法，都不能被实例化（不能创建对象）。

3.2 接口、抽象类、类之间的关系

（1）接口是抽象方法的集合，如果一个类实现了某个接口，那么它就继承了这个接口的抽象方法。

（2）类与接口是实现关系，而且是多实现，一个类可以实现多个接口。

（3）类与抽象类是继承关系，Java中不支持多继承，但支持多重继承。

3.3 接口、抽象类、类之间的区别

（1）抽象类可以有成员变量，也可以有常量； 接口只能有常量，默认修饰符public static final。

（2）抽象类可以有抽象方法，也可以有非抽象方法； 接口只能有抽象方法，默认修饰符 public abstract。

（3）抽象类有构造方法，为子类提供； 接口没有构造方法。

5.扩展思考

什么是多态？

多态，简单的理解就是事物的多种形态。专业的术语说就是：同一个实现接口，使用不同的实例而执行不同的操作。 Java作为一种面向对象的语言其多态性体现在一个子类对象(实现类对象)既可以给这个子类(实现类对象)引用变量赋值， 又可以给这个子类(实现类对象)的父类(接口)变量赋值。如Student类是Person类的子类， 那么一个Student对象既可以赋值给一个Student类型的引用，也可以赋值给一个Person类型的引用。 最终多态体现为父类引用变量可以指向子类对象。 多态的前提是必须要有子父类（继承）关系。 在使用多态后的父类引用变量调用方法时，会调用子类重写后的方法。

7.更多讨论

7.1 什么时候使用抽象类和接口？

答：如果你想实现多重继承，那么就必须使用接口。由于Java不支持多继承，子类不能够继承多个类，但可以实现多个接口。因此就可以用接口来解决它。 如果基本功能在不断改变，那么就需要使用抽象类。如果不断改变基本功能并且使用接口，那么就需要改变所有实现了该接口的类。

7.2 已经有抽象类了，为什么还要使用接口？

答：因为接口是可以多继承的。 还有一点也是最重要的一点本质的一点"抽象类是对象的抽象，然而接口是一种行为规范"。

7.3 一个接口只能有一个实现类吗？

答：接口可以有无数的实现类，只要这个类在声明的时候表示implement某个接口。
